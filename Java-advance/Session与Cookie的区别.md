# Session与Cookie的区别

**基本概念**

含义：Session代表一次用户会话，即从客户端浏览器连接服务器开始，到客户端浏览器与服务器断开为止

作用：Session范围内的属性可以在一次会话中的多个页面的跳转之间共享

应用：跟踪用户的会话信息，跟踪用户购买的商品



为什么要Cookie？

由于Http是一种无状态的协议，当用户的一次访问请求结束之后，后端服务器就无法知道下一次来访问的还是不是上次访问的用户

作用：服务端可以根据客户端发出的Cookie值来区分不同的用户（简单的说就是判断发出请求的客户的身份）。

应用：记录客户端的用户名喜好等

**区别**：

- Session会随着浏览器的关闭而失效，但Cookie会一直存放在客户端的机器中，除非超出了Cookie的生命周期
- Session的安全性要比Cookie高，原因是Session将数据保存在服务端，只通过Cookie传递一个SessionID而已，而Cookie将所有数据都存储在客户端的浏览器里《深入分析javaweb技术内幕——许令波》

**补充：**web服务器跟踪客户状态的四种方法（Http是无状态的协议）

1. 在HTML表单中加入隐藏字段，它包含用于跟踪客户状态的数据
2. 重写URL，使它包含跟踪客户状态的数据
3. 使用Cookie来传送用于跟踪客户状态的数据
4. 使用会话Session机制
