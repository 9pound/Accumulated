# Collection、List、Set、Map、Queue、Stack

**Collection**

- 本质：一个独立的的元素序列，这些元素都服从一条或多条规则。
- （序列——一种存放对象的方式）

- 作用：描述所有序列容器的根接口（附属接口）
- 附属接口：即因为要表示其他若干接口的共性而出现的接口

**List**

- List是一个有序集合，也叫它表，
- 特点：按照插入的顺序保存元素、允许有重复元素

**Set**

- Set是一个无序集合
- 特点：不允许有重复元素
- 就是一个Collection，只不过方法的行为有更严格的定义

**Map**

- 本质：一组成对的“键值对”对象，允许你使用键来查找值。
- 可以看做是一个特殊的数组，每一个插入其中的元素，其位置都是确定的。
- 别名："映射表"，“关联数组”，“字典”。

**Queue**

- 本质：是一个先进先出（FIFO）的容器
- 特点：按照排队规则来确定对象产生的顺序（通常与元素被查如的顺序相同）


**Stack**

- 本质：是一种先进后出（LIFO）的容器，也被称作叠加栈
